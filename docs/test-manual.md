# PNR Demo 测试手册

## 测试入口

- 页面：CDK 输出的 `WebUrl`，形如 `http://<web-instance-public-dns>`。
- 操作：输入测试语句或点击示例按钮后，再点击“生成 PNR 指令”。
- 期望：结果区出现 PNR 指令；无法生成时，应看到明确的补充/错误提示。
- 接口直测需带 Cognito JWT（`authorization: Bearer <IdToken>`），且令牌所属用户已加入对应组（`translator`/`executor`/`reviewer`）。

## 1. 航班查询成功路径

| 场景 | 测试语句 | 期望 |
| --- | --- | --- |
| 单程指定日期时间航司 | 下个月23号13点50北京到成都中转上海东航 | 生成类似 `av:m/PEKCTU/23jul26/1350/MU` |
| 单程直飞 | 查7月25号上午北京到上海东航直飞航班 | 生成包含 `/d` 的指令 |
| 单程无经停 | 查7月25号北京到上海无经停航班 | 生成包含 `/n` 的指令 |
| 按到达时间排序 | 查7月25号北京到上海按到达时间排序 | 生成 `av:a/...` |
| 完整显示 | 查7月25号北京到上海完整航班 | 生成 `av:h/...` |
| 承运方显示 | 查7月25号北京到上海承运航班 | 生成 `av:o/...` |
| 中转显示 | 查7月25号北京到成都中转上海航班 | 生成 `av:m/...` |
| 往返 | 查一下7月20号上午北京到上海，7月23号晚上上海回北京的东航往返航班 | 生成包含 `&` 的往返指令 |
| 多段 | 帮我查7月20号北京到上海，7月22号上海到成都的多段航班 | 生成包含 `&` 的多段指令 |

## 2. PNR 低风险操作

| 场景 | 测试语句 | 期望 |
| --- | --- | --- |
| 添加素食餐 | 给第一个旅客第二段加一个东航素食餐 | 生成 `SSR VGML MU HK1/P1/S2` |
| 添加儿童餐 | 给第一个旅客第一段加一个东航儿童餐 | 生成 `SSR CHML MU HK1/P1/S1` |
| 添加婴儿餐 | 给第一个旅客第一段加一个东航婴儿餐 | 生成 `SSR BBML MU HK1/P1/S1` |
| 添加联系电话 | 给旅客补一个联系电话 13800138000 | 预览为脱敏版 `CTCM PHONE_1`；真实指令 `CTCM 13800138000` 仅写入加密安全存储，不在 API 响应出现 |

## 3. 高风险/确认路径

| 场景 | 测试语句 / 操作 | 期望 |
| --- | --- | --- |
| 取消航段 | 取消第二段航段 | 风险等级为高，状态 `manual_review`；不自动执行，需人工审核 |
| 取消不存在航段 | 取消第三段航段 | 应提示无法定位或需要补充 |
| 执行确认 | 先生成任一可执行指令，再点“确认执行” | 执行反馈显示已提交/已执行 |

## 4. 审核与权限闭环（接口测）

| 场景 | 操作 | 期望 |
| --- | --- | --- |
| 高风险走审核 | 取消航段得到 `manual_review` → 由 `reviewer` 组用户调用 `/approve` | 翻译状态转为 `ready_for_confirm`，记录 `reviewed_by`/`reviewed_at`，之后可执行 |
| maker-checker | 由发起翻译的同一用户调用 `/approve` | 返回 403，提示 maker-checker 冲突（审核人不能是发起人）|
| 审核驳回 | `reviewer` 调用 `/reject` | 翻译状态转为 `rejected`，不可执行 |
| 缺少角色 | 用无任何组的 JWT 调用 `/translate` | 返回 403（无 fail-open 默认角色）|
| 越权执行 | 仅 `translator` 角色调用 `/execute` | 返回 403（需 `executor`/`admin`）|
| 幂等执行 | 用相同 `idempotency_key` 重复调用 `/execute` | 两次返回同一 `execution_id`，不重复执行、不报错 |
| 过期上下文 | `/execute` 传与翻译时不一致的 `pnr_context_version` | 返回错误 `stale pnr_context_version` |

## 5. 需补充信息

| 场景 | 测试语句 | 期望 |
| --- | --- | --- |
| 缺日期 | 查北京到上海航班 | 提示需要补充日期 |
| 缺到达地 | 查7月25号北京出发的航班 | 提示需要补充到达地 |
| 缺旅客/航段 | 加一个素食餐 | 提示需要补充旅客或航段 |
| 相对星期 | 查下周五北京到上海上午东航直飞航班 | 当前可能提示需要补充具体日期 |

## 6. 边界和错误输入

| 场景 | 测试语句 | 期望 |
| --- | --- | --- |
| 未知城市 | 查7月25号火星到上海航班 | 提示需要补充或无法识别地点 |
| 未知航司 | 查7月25号北京到上海某某航空航班 | 不应生成错误航司代码 |
| 日期格式异常 | 查13月40号北京到上海航班 | 不应生成有效指令 |
| 空输入 | 清空输入后点击生成 | 显示友好错误 |
| 长口语输入 | 客人想下个月23号下午从北京去成都，中间在上海转一下，最好东航 | 应尽量生成单程查询指令 |

## 7. 回归检查

每次改动后至少跑：

```bash
npm test
npm run build
node --check web/app.js
```

页面只改静态文件时，用 Cognito `IdToken` 跑：

```bash
PNR_ID_TOKEN="<your-cognito-id-token>" npm run deploy:web
```

不要为了页面改动运行 `npm run deploy`，避免替换 EC2。
